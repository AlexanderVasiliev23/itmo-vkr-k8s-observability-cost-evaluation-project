# Запуск экспериментов на runner VM

## Зачем нужна runner VM

Длительные эксперименты могут занимать от нескольких часов до суток. Запуск с локальной машины неудобен: соединение может оборваться, ноутбук может быть выключен, а окружение — измениться.

Runner VM — выделенная виртуальная машина в облаке, на которой запускается ObsBench. Она позволяет выполнять долгие прогоны в стабильном окружении и при необходимости отсоединяться от сессии.

## Общая схема

```text
локальная машина
  ├── подготавливает kubeconfig
  ├── собирает бинарный файл ObsBench
  └── копирует артефакты на runner VM

runner VM
  ├── имеет доступ к Kubernetes-кластеру
  ├── запускает ObsBench в tmux
  └── сохраняет результаты экспериментов
```

## Требования

На runner VM должны быть доступны:

- `kubectl`
- `yc` CLI
- Docker
- `tmux`
- kubeconfig для доступа к кластеру
- бинарный файл `obs-bench`
- конфигурационный файл экспериментов

## Пример запуска

```bash
ssh ubuntu@<RUNNER_IP>

tmux new -s bench

KUBECONFIG=~/.kube/k1.yaml ./obs-bench run-batch --file experiments-final.yaml
```

Отсоединиться от tmux-сессии:

```text
Ctrl+B, затем D
```

Вернуться в сессию:

```bash
tmux attach -t bench
```
